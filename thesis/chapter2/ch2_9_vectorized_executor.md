# 2.9. Векторизованный исполнитель (Vectorized Executor)

Для преодоления ограничений интерпретатора Python, в аналитических базах данных часто используется векторизованная (или колоночная) модель выполнения. В основе векторизации лежит отказ от передачи данных по одной строке (кортежу). Вместо этого операторы обмениваются "пакетами" (batches), состоящими из колонок.

В Python эта парадигма идеально ложится на библиотеку **NumPy**, которая написана на C и Fortran, что позволяет обходить GIL (Global Interpreter Lock) и использовать SIMD-инструкции процессора.

### 2.9.1. Базовый интерфейс NumPy-Батчинга

Класс `VectorizedOperator` (`executor/vectorized_operators.py`) меняет сигнатуру итератора. Вместо `__next__() -> Dict[str, Any]` он использует `next_batch(batch_size: int = 1024) -> Optional[Dict[str, np.ndarray]]`.

```python
class VectorizedOperator:
    # Возвращает словарь, где ключ - имя колонки, а значение - массив NumPy (n элементов)
    def next_batch(self, batch_size: int = 1024) -> Optional[Dict[str, np.ndarray]]:
        raise NotImplementedError
```
*Интуиция данных*: Эта структура данных известна как "Кэш-линия дружественная" (Cache-line friendly). Если процессору нужно сложить колонки `price` и `quantity`, он загружает их в L1-кэш непрерывным блоком, выполняя математику мгновенно, в отличие от построчной модели, где процессору приходилось бы делать случайные прыжки по оперативной памяти для извлечения свойств объектов-словарей.

### 2.9.2. Фильтрация через булевы маски (Boolean Masking)

Для оценки производительности векторизации лучше всего рассмотреть оператор фильтрации (`VectorizedFilter`). В `Volcano`-модели мы проверяли каждую строку через `if`. В векторизованном движке цикла `if` нет вообще:

```python
class VectorizedFilter(VectorizedOperator):
    def next_batch(self, batch_size: int = 1024) -> Optional[Dict[str, np.ndarray]]:
        # Запрашиваем 1000 строк одним куском
        batch = self.input.next_batch(batch_size)
        if batch is None: return None

        # Оцениваем предикат над всем массивом (CPython вызывает C-код NumPy)
        # mask - это массив булевых значений (True/False)
        mask = self.evaluate(self.condition, batch)
        
        if np.any(mask):
            # Мгновенная выборка (fancy indexing) только нужных строк по маске
            filtered_batch = {k: v[mask] for k, v in batch.items()}
            return filtered_batch
```
*Кодовый алгоритм*: Метод `self.evaluate(self.condition, batch)` возвращает массив типа `[True, False, True...]`. Затем конструкция `v[mask]` за одну миллисекунду оставляет в батче только те строки, где стоит `True`. Таким образом, Python используется исключительно как дирижер, отправляющий тяжелую работу в скомпилированные низкоуровневые библиотеки.

Именно этот подход позволяет движку обрабатывать миллионы строк в секунду (как будет продемонстрировано в тестах), что зачастую превосходит показатели наивно написанного C++ кода.

# 2.10. Реляционные операторы: Join и Aggregation

Реализация "тяжелых" реляционных операторов, таких как склейка таблиц (JOIN) и агрегация (GROUP BY), — задача, определяющая предел масштабирования движка. Наивный подход через циклы имеет сложность $O(N \times M)$ и не способен обработать больше нескольких тысяч строк. В нашем решении используются алгоритмы на базе хеш-таблиц и бинарного поиска.

### 2.10.2. Векторная агрегация (GROUP BY)
Оператор `VectorizedHashAggregate` работает по схожему принципу. Он должен разбить входящий пакет на группы и вычислить показатели вроде `SUM()` или `COUNT()`. 
* В Python это делалось бы через `defaultdict(int)`. 
* В NumPy мы используем `np.unique` для мгновенного поиска всех уникальных групп, и функцию `np.add.at`, которая атомарно суммирует значения одинаковых групп напрямую в С-массиве:

```python
# Поиск уникальных групп и их обратных индексов
unique_keys, inverse = np.unique(group_cols[0], return_inverse=True)

# ... логика инстанцирования пустого массива ...
if agg.func == "SUM": 
    # Нативна С-агрегация: прибавить к результатам по индексам (inverse) значения выражений
    np.add.at(reduced_array, inverse, expr_data)
```
Именно эти низкоуровневые трюки с индексацией позволяют нашему движку показывать результаты, которые будут проанализированы в разделе бенчмарков.


# Mini SQL Engine

Высокопроизводительный модульный процессор SQL-запросов, написанный с нуля на Python. Этот движок демонстрирует полный жизненный цикл SQL-запроса: от парсинга необработанной строки до высокоскоростного векторизованного выполнения с использованием NumPy.

---

## Архитектура и конвейер (Pipeline)

Движок спроектирован согласно современной архитектуре баз данных и разделен на этап логического планирования и несколько стратегий физического выполнения.

### 1. Парсинг SQL (`parser/`)
- **Движок**: Работает на базе [Lark](https://github.com/lark-parser/lark).
- **Грамматика**: Кастомная EBNF-грамматика с поддержкой `SELECT`, `JOIN` (Inner/Left), `WHERE`, `GROUP BY`, `HAVING` и `ORDER BY`.
- **AST**: Строго типизированное абстрактное синтаксическое дерево, построенное на Python `dataclasses`.

### 2. Семантический анализ и привязка (`planner/binder.py`)
- Сопоставляет идентификаторы (таблицы, столбцы) с данными из **Системного каталога** (System Catalog).
- Выполняет проверку типов и привязывает выражения к физическим схемам таблиц.
- Обрабатывает псевдонимы (aliasing) и разрешения пространств имен для сложных JOIN-запросов.

### 3. Логическая оптимизация (`planner/optimizer.py`)
**Rule-Based Optimizer (RBO)** — оптимизатор на основе правил, который трансформирует логический план для максимальной эффективности:
- **Predicate Pushdown**: Сдвигает фильтры максимально близко к источнику данных.
- **Constant Folding**: Вычисляет статические выражения (например, `5 + 10`) на этапе компиляции.
- **Column Pruning**: Исключает неиспользуемые столбцы из сканирования для минимизации ввода-вывода.
- **Index Selection**: Автоматически переключается на `IndexScan`, если фильтр равенства совпадает с первичным ключом.

### 4. Движки выполнения (`executor/`)
Поддерживаются две различные модели выполнения:
- **Volcano Model (Row-based)**: Классический итераторный подход (`next()`) для потоковой обработки данных с минимальными затратами памяти.
- **Vectorized Engine (Batch-based)**: Высокопроизводительный движок, обрабатывающий данные пакетно по столбцам с помощью **NumPy**, что практически устраняет накладные расходы Python на обработку каждой строки.

## Производительность

Движок рассчитан на масштабируемость. На наборе данных в **1 000 000 строк** векторизованный движок зачастую превосходит стандартную итерационную модель в **10–50 раз**.

### Бенчмарки
Вы можете запустить демонстрацию производительности, чтобы увидеть движок в деле:
```bash
python -m benchmarks.final_showcase
```

Скрипт генерирует:
1. ASCII-диаграмму прямо в терминале для мгновенного сравнения.
2. Детальный визуальный отчет: `reports/benchmark_chart.png`.

## Быстрый старт

### 1. Установка зависимостей
```bash
pip install lark numpy matplotlib
```

### 2. Запуск демо
Изучите процесс парсинга и генерации AST:
```bash
python -m parser.parser
```

### 3. Стресс-тестирование
Проверьте лимиты движка на 1 миллионе строк:
```bash
python -m benchmarks.stress_test
```

## Структура проекта
- `parser/`: Определение грамматики и построение AST.
- `planner/`: Binder, логический оптимизатор и физические планировщики.
- `executor/`: Реализация операторов Volcano и Vectorized.
- `catalog/`: Метаданные таблиц и управление схемами.
- `benchmarks/`: Скрипты для тестирования производительности и наборы данных.
- `visualization/`: Инструменты для визуализации планов выполнения и результатов бенчмарков.

---

## Философия проектирования
- **Модульность**: Каждый этап конвейера изолирован и легко расширяем.
- **Приоритет производительности**: Использование NumPy для массовых операций с данными в обход медленных циклов Python.
- **Расширяемость**: Архитектура RBO позволяет легко добавлять новые правила оптимизации (например, переупорядочивание Join-ов).
